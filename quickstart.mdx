## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** v18 or later
- **MongoDB** running locally (or a remote MongoDB URI)
- **npm** (bundled with Node.js)

<Steps>
  <Step title="Clone or download the project">
    Download the project source and navigate into the directory:

    ```bash
    git clone <your-repo-url> sps-school-backend
    cd sps-school-backend
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages:

    ```bash
    npm install
    ```

    This installs the following runtime dependencies:

    | Package | Version | Purpose |
    |---|---|---|
    | `express` | ^5.2.1 | HTTP server framework |
    | `mongoose` | ^9.3.0 | MongoDB ODM |
    | `jsonwebtoken` | ^9.0.3 | JWT issuance and verification |
    | `bcryptjs` | ^3.0.3 | Password hashing |
    | `dotenv` | ^17.3.1 | Environment variable loading |
    | `cors` | ^2.8.6 | Cross-origin request handling |

    `nodemon` is installed as a dev dependency for auto-restarting during development.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root with the following variables:

    ```bash
    PORT=5000
    DB_URL=mongodb://127.0.0.1:27017/sps_school
    JWT_SECRET=mysecretkey
    ```

    <Warning>
      Replace `JWT_SECRET` with a long, random string before deploying to any non-local environment. The value shown above is for local development only.
    </Warning>

    | Variable | Description |
    |---|---|
    | `PORT` | The port the Express server listens on |
    | `DB_URL` | MongoDB connection string used by Mongoose |
    | `JWT_SECRET` | Secret key used to sign and verify JWT tokens |
  </Step>

  <Step title="Start the server">
    <Tabs>
      <Tab title="Production / standard">
        ```bash
        node server.js
        ```
      </Tab>
      <Tab title="Development (auto-reload)">
        ```bash
        npx nodemon server.js
        ```

        <Tip>
          Add a `dev` script to `package.json` for convenience:

          ```json
          "scripts": {
            "dev": "nodemon server.js"
          }
          ```

          Then run `npm run dev`.
        </Tip>
      </Tab>
    </Tabs>

    When the server starts successfully you will see:

    ```
    Database connected
    Server running on port 5000
    ```
  </Step>

  <Step title="Verify the server is running">
    Send a request to the root endpoint:

    ```bash
    curl http://localhost:5000/
    ```

    Expected response:

    ```
    SPS School Backend Running
    ```
  </Step>

  <Step title="Register your first user">
    Create a user account by posting to the register endpoint:

    ```bash
    curl -X POST http://localhost:5000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Alice Smith",
        "email": "alice@example.com",
        "password": "securepassword",
        "role": "SUPER_ADMIN"
      }'
    ```

    The server responds with the created user document (including the hashed password).
  </Step>

  <Step title="Log in and retrieve a token">
    ```bash
    curl -X POST http://localhost:5000/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "alice@example.com",
        "password": "securepassword"
      }'
    ```

    Expected response:

    ```json
    {
      "token": "<jwt-token>",
      "role": "SUPER_ADMIN"
    }
    ```

    Use the returned `token` value as the `authorization` header on protected requests.
  </Step>
</Steps>
